Ansible inventory plugins vs scripts

Architecture comparison, annotated code, step-by-step plugin guide, and a decision framework for when to use each.

Inventory scriptlegacy
  • Ansible forks a child process and calls the script via the OS
  • Called at least twice: once with --list, once with --host <name> per host unless _meta is used
  • Any language (Python, bash, Ruby) — but no access to Ansible internals
  • No native caching — you must implement it yourself
  • No keyed_groups, compose, or filters — all grouping logic hand-built
  • Marked as deprecated — no new features being added
  • Still works — maintained for backward compatibility only
# How Ansible calls a script: ./inventory.py --list # → JSON blob of all groups + hosts ./inventory.py --host web-01 # → JSON of host vars (one call per host!) # Unless _meta.hostvars is in --list output
Inventory pluginmodern
  • Runs as a Python class inside Ansible's own process — no subprocess overhead
  • Called once — the plugin populates the inventory object directly in memory
  • Full access to Ansible internals: display, templar, config system
  • Native caching built into the plugin base class — one line to enable
  • Native keyed_groups, compose, filters — configured in YAML, no code needed
  • All active development is here — all cloud plugins are plugins
  • Requires Python — cannot be written in bash or Ruby
# How Ansible loads a plugin: # 1. Reads aws_ec2.yml # 2. Matches filename → selects plugin class # 3. Calls plugin.parse(inventory, loader, path) # 4. Plugin populates inventory in-process # No subprocess. One call. Done.
The performance difference matters at scale. A script with 200 hosts and no _meta makes 201 subprocess calls. A plugin makes one in-process call regardless of fleet size.
Required JSON contract — --listscript
# ./inventory.py --list must return: { "webservers": { "hosts": ["web-01", "web-02"], "vars": { "nginx_port": 80 } }, "databases": { "hosts": ["db-01"], "vars": {} }, "_meta": { ← critical optimisation "hostvars": { "web-01": { "ansible_host": "10.0.1.10", "ec2_id": "i-abc123" }, "web-02": { "ansible_host": "10.0.1.11" }, "db-01": { "ansible_host": "10.0.2.10" } } } }
  • Including _meta.hostvars tells Ansible all host vars are here — skips --host <name> calls entirely
  • Omit _meta and Ansible calls --host once per host — N+1 subprocess calls
Required JSON contract — --hostscript
# ./inventory.py --host web-01 must return: { "ansible_host": "10.0.1.10", "ansible_user": "ubuntu", "ec2_region": "eu-west-2" } # If _meta was in --list, return empty dict: {} # Minimal working script structure: import json, sys if '--list' in sys.argv: print(json.dumps(build_inventory())) elif '--host' in sys.argv: # return empty — _meta handles it print(json.dumps({})) else: print(json.dumps({}))
  • Both --list and --host must be handled or Ansible throws an error
  • The script must exit with code 0 on success, non-zero on failure
Plugin class structureplugin
# inventory_plugins/mycompany.py from ansible.plugins.inventory import \ BaseInventoryPlugin, Constructable, Cacheable # BaseInventoryPlugin — core plugin machinery # Constructable — adds keyed_groups, compose # Cacheable — adds built-in cache support class InventoryModule( BaseInventoryPlugin, Constructable, Cacheable ): # Must match the filename (without .py) NAME = 'mycompany' def verify_file(self, path): # Return True only for files this plugin handles valid = False if super().verify_file(path): if path.endswith(( 'mycompany.yml', 'mycompany.yaml' )): valid = True return valid def parse(self, inventory, loader, path, cache=True): # Main entry point — called once by Ansible super().parse(inventory, loader, path) self._read_config_data(path) # Use cache if available and not expired cache_key = self.get_cache_key(path) if cache and cache_key in self._cache: data = self._cache[cache_key] else: data = self._fetch_from_api() self._cache[cache_key] = data self._populate(data)
Populating inventory + native featuresplugin
def _populate(self, data): for host in data['hosts']: name = host['name'] # Add host to inventory self.inventory.add_host(name) # Set host variables self.inventory.set_variable( name, 'ansible_host', host['ip'] ) self.inventory.set_variable( name, 'datacenter', host['dc'] ) self.inventory.set_variable( name, 'role', host['role'] ) # Add to a manual group group = 'role_' + host['role'] self.inventory.add_group(group) self.inventory.add_child(group, name) # Apply keyed_groups from YAML config — free! self._add_host_to_keyed_groups( self.get_option('keyed_groups'), self.inventory.get_host(name).get_vars(), name, strict=False ) # Apply compose from YAML config — free! self._set_composite_vars( self.get_option('compose'), self.inventory.get_host(name).get_vars(), name, strict=False )
  • Calling _add_host_to_keyed_groups gives your plugin the same keyed_groups: YAML config as aws_ec2 — for free
  • Calling _set_composite_vars gives your plugin compose: support — also free
step 1Project layout for a custom plugin▾
ansible-project/ inventory_plugins/ mycompany.py ← your plugin file inventories/ production/ mycompany.yml ← config file (triggers the plugin) group_vars/ all/main.yml ansible.cfg
  • Point Ansible to your local plugin directory in ansible.cfg
  • The config file name must match the plugin's NAME attribute
step 2Register the plugin in ansible.cfg▾
# ansible.cfg [defaults] # Add your local plugin directory to the search path inventory_plugins = ./inventory_plugins # Enable your plugin by name [inventory] enable_plugins = mycompany, host_list, yaml, ini, auto
  • Always include host_list, yaml, ini, auto alongside your plugin — omitting them breaks static inventory files
step 3DOCUMENTATION block — required for ansible-doc▾
# mycompany.py — at the top of the file DOCUMENTATION = r''' name: mycompany short_description: Pulls inventory from internal CMDB API description: - Connects to CMDB and returns hosts grouped by role. options: api_url: description: Base URL of the CMDB API required: true type: str keyed_groups: description: Add hosts to groups based on variable values compose: description: Create vars from Jinja2 expressions notes: - Set CMDB_TOKEN env var for authentication. ''' EXAMPLES = r''' # inventories/production/mycompany.yml plugin: mycompany api_url: https://cmdb.internal/api keyed_groups: - key: role prefix: role compose: ansible_host: ip_address '''
  • DOCUMENTATION is required — without it ansible-doc -t inventory mycompany fails
  • Options declared here are automatically validated against the config YAML — clear errors on misconfiguration
step 4Test and validate▾
# Verify the plugin is discoverable: ansible-doc -t inventory mycompany # Parse inventory with your plugin: ansible-inventory -i inventories/production/ --list # See the group tree: ansible-inventory -i inventories/production/ --graph # Enable debug logging to trace plugin execution: ansible-inventory -i inventories/production/ --list -vvv # Test connectivity against a group: ansible -i inventories/production/ role_webserver -m ping
Use a script when...script
  • You're maintaining an existing script that works and a rewrite isn't justified
  • You need inventory logic in a language that isn't Python (rare but valid)
  • The query is a one-liner that would be over-engineered as a full plugin class
  • You have a script from before Ansible 2.4 that predates plugins
  • Never choose a script for new development — accumulates tech debt immediately
Use a plugin when...default choice
  • You're building anything new — plugins are the only right answer for new custom inventory sources
  • You want keyed_groups, compose, or caching without writing it yourself
  • You want ansible-doc support and self-documenting options
  • Performance matters — in-process beats subprocess at any scale
  • You want Ansible's config validation to catch misconfigured options early
  • Requires Python — the only real constraint
If you're migrating an existing script to a plugin: the _populate() method is where your script's logic goes. The rest of the plugin class is boilerplate. A typical migration takes an afternoon and immediately gains caching and keyed_groups support.
