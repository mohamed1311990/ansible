AWS EC2, Azure, GCP, and custom scripts — annotated plugin configs, setup steps, debugging commands, and caching patterns.

setupPrerequisites▾
  • Install the collection: ansible-galaxy collection install amazon.aws
  • Install Python deps: pip install boto3 botocore
  • AWS credentials via env vars, ~/.aws/credentials, or IAM instance role
  • Required IAM permissions: ec2:DescribeInstances, ec2:DescribeTags
configaws_ec2.yml — the inventory plugin file▾
  • File MUST end in aws_ec2.yml or aws_ec2.yaml — Ansible uses the filename to select the plugin
  • Place it inside your inventory directory: inventories/production/aws_ec2.yml
--- # inventories/production/aws_ec2.yml plugin: amazon.aws.aws_ec2 # Which regions to query regions: - eu-west-2 - us-east-1 # Only return running instances filters: instance-state-name: running tag:Environment: production # filter by tag # Use private IP (for VPC / bastion setups) hostnames: - private-ip-address # Auto-group by EC2 tags and attributes keyed_groups: - key: tags.Role # tag Role=webserver → group role_webserver prefix: role - key: tags.Environment prefix: env - key: placement.region # auto-group by region prefix: aws_region # Expose EC2 metadata as host vars compose: ansible_host: private_ip_address ec2_instance_type: instance_type ec2_region: placement.region
  • keyed_groups auto-creates Ansible groups from EC2 tags — tag an instance Role=webserver and it lands in group role_webserver automatically
  • compose maps EC2 metadata fields to Ansible host variables
  • Avoid public-ip-address as hostname in production — use private IP with a bastion or VPN
usageUsing the dynamic inventory▾
# Verify what hosts are returned: ansible-inventory -i inventories/production/ --graph # Output (groups auto-created from tags): @all: |--@role_webserver: | |--10.0.1.10 | |--10.0.1.11 |--@role_database: | |--10.0.2.10 |--@env_production: | |--10.0.1.10 | |--10.0.2.10 # Run a playbook against a tag-based group: ansible-playbook -i inventories/production/ \ playbooks/deploy.yml \ --limit role_webserver
  • Mix static and dynamic inventory in the same directory — Ansible merges them automatically
  • Add a static group_vars/role_webserver/main.yml and it applies to all EC2 instances with Role=webserver tag

configazure_rm.yml — the inventory plugin file▾
  • File MUST end in azure_rm.yml or azure_rm.yaml
--- # inventories/production/azure_rm.yml plugin: azure.azcollection.azure_rm # Filter by resource group include_vm_resource_groups: - production-rg - shared-rg # Auth via service principal (env vars) auth_source: env # Env vars needed: # AZURE_CLIENT_ID, AZURE_SECRET # AZURE_SUBSCRIPTION_ID, AZURE_TENANT # Group by tags keyed_groups: - prefix: role key: tags.role | default('ungrouped') - prefix: env key: tags.environment | default('ungrouped') # Use private IP use_private_ip: true # Expose Azure metadata as vars compose: ansible_host: private_ipv4_addresses[0] azure_vm_size: vm_size azure_location: location
  • Service principal auth is recommended for CI — set the four env vars as pipeline secrets
  • | default('ungrouped') prevents errors when a VM is missing the expected tag

configgcp_compute.yml — the inventory plugin file▾
--- # inventories/production/gcp_compute.yml plugin: google.cloud.gcp_compute projects: - my-gcp-project-id zones: - europe-west2-a - us-central1-a # Auth via service account file auth_kind: serviceaccount service_account_file: ~/.gcp/service-account.json # Filter to running instances only filters: - status = RUNNING # Group by GCP labels (equivalent to tags) keyed_groups: - key: labels.role prefix: role - key: labels.environment prefix: env - key: zone prefix: gcp_zone compose: ansible_host: networkInterfaces[0].networkIP
  • GCP uses "labels" instead of "tags" — same concept, different field name in the plugin config
  • Never commit a path to a real service account JSON key — use a secrets manager or env var in CI
conceptScript vs plugin — what's the difference?▾
old wayInventory scripts
  • Executable script (any language) called by Ansible
  • Must respond to --list with JSON of all hosts
  • Must respond to --host <name> with host vars
  • Deprecated — being phased out in favor of plugins
  • Slower — Ansible calls the script multiple times
modernInventory plugins
  • Python class implementing the InventoryPlugin API
  • Configured via a *.yml file — no CLI flags needed
  • Runs inside Ansible's process — faster, more integrated
  • Supports caching, compose, keyed_groups natively
  • The right approach for all new custom inventory sources
exampleMinimal custom inventory script (legacy pattern)▾
  • Useful when querying an internal CMDB, ServiceNow, or custom API with no existing plugin
  • Must be executable: chmod +x inventory.py
#!/usr/bin/env python3 # inventory.py — minimal custom script import json, sys, requests def get_hosts(): # Query your internal API / CMDB r = requests.get('https://cmdb.internal/api/hosts') hosts = r.json() inventory = { 'webservers': {'hosts': [], 'vars': {'nginx_port': 80}}, 'databases': {'hosts': []}, '_meta': {'hostvars': {}} } for host in hosts: group = host['role'] + 's' inventory[group]['hosts'].append(host['name']) inventory['_meta']['hostvars'][host['name']] = { 'ansible_host': host['ip'], 'datacenter': host['dc'] } return inventory if __name__ == '__main__': if '--list' in sys.argv: print(json.dumps(get_hosts())) elif '--host' in sys.argv: print(json.dumps({})) # use _meta instead
  • Always populate _meta.hostvars — avoids Ansible calling --host once per host (huge speed win for large inventories)
  • Usage: ansible-playbook -i ./inventory.py playbooks/site.yml
debuggingEssential debugging commands▾
# See all hosts returned by dynamic inventory: ansible-inventory -i inventories/production/ --list # See the group tree: ansible-inventory -i inventories/production/ --graph # Inspect one host's resolved vars: ansible-inventory -i inventories/production/ --host 10.0.1.10 # Test connectivity against a dynamic group: ansible -i inventories/production/ role_webserver -m ping # See what the plugin is doing (verbose): ansible-inventory -i inventories/production/ --list -vvv
performanceCaching — avoid hitting the API on every run▾
  • Every playbook run queries the cloud API — slow for large fleets, may hit rate limits
  • Enable caching in ansible.cfg to cache results for N seconds
# ansible.cfg [inventory] cache = True cache_plugin = jsonfile # or redis, memcached cache_timeout = 300 # seconds (5 minutes) cache_connection= /tmp/ansible-inv-cache # Force a cache refresh when needed: ansible-inventory --list --refresh-cache
  • Use jsonfile for local dev, redis for shared CI runners where multiple jobs need the same inventory
  • 5-minute cache is fine for most use cases — don't cache too long if your fleet scales frequently
patternMixing static + dynamic in one inventory directory▾
  • Ansible merges all files in the inventory directory automatically
  • Use static files for hosts that never change (bastion, VPN gateway) alongside dynamic EC2 inventory
  • Static group_vars/ files apply to dynamic groups — name the directory after the tag-generated group name
inventories/production/ aws_ec2.yml # dynamic — queries EC2 API hosts.yml # static — bastion, fixed hosts group_vars/ all/ main.yml vault.yml role_webserver/ # matches EC2 tag Role=webserver main.yml # applies to all webserver EC2 instances role_database/ main.yml
The group name role_webserver comes from the keyed_groups config: prefix: role + key: tags.Role + tag value webserver → role_webserver. Name your group_vars/ directories to match this exactly.
